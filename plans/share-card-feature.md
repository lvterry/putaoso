# Variety share card feature · implementation plan

> Version: v1 · 2026-05-28 · Status: product direction locked

## Goal

Add a share button on each live variety detail page, near the variety name. When clicked, the page opens a modal dialog containing a polished share card for that variety. The user can download the card as a PNG file.

This feature should feel native to Putaoso: a collectible field-guide card for wine grapes. It should be beautiful enough for social media sharing, but still clearly informational rather than decorative only.

## Current Understanding

- Site is a fully static Astro site.
- Detail pages are generated by `src/pages/[slug].astro`.
- Variety data already exists in `src/content/varieties/*.md` frontmatter.
- There is no client framework; interactivity should use vanilla browser APIs unless there is a strong reason to add a small dependency.
- Existing CSS is centralized in `src/styles/global.css`.
- The share button should appear near:
  - `name_en`
  - `name_cn`
  - aliases / quote area

## Requirements Interview

Answered direction:

- Primary use: social sharing across WeChat, Xiaohongshu, Instagram, and similar platforms.
- Card concept: collectible field-guide card.
- Format: one fixed `1080 × 1350` portrait PNG for v1.
- Content density: mostly name, quote, flavors, and branding.
- Dependency policy: okay to add a small client-side export dependency.
- Button label: `分享`.
- Sharing flow: keep v1 download-only; skip Web Share API until a later iteration.
- Visual connection: reuse the existing variety SVG scene, or at minimum derive a scene-connected visual accent from it.

### 1. Share Card Purpose

1. Target destinations:
   - WeChat
   - Xiaohongshu
   - Instagram
   - other social platforms / camera-roll sharing

2. Primary purpose:
   - collectible “grape encyclopedia” / field-guide card
   - attractive enough to share
   - light social preview back to the site

3. V1 sharing behavior:
   - Download-only is enough.
   - Do not add Web Share API yet.

### 2. Card Format

1. Target output:
   - `1080 × 1350` portrait PNG

2. v1 ships one fixed format.

3. Recommended background:
   - full designed background, not transparent
   - use intentional light paper tones so exported cards are stable across light/dark system settings

### 3. Card Content

V1 should stay mostly name, quote, flavors, and branding.

Recommended fields:

- Variety number, e.g. `N° 003`
- English name
- Chinese name
- Type: red / white / rosé
- Origin
- Hero quote or `card_tagline`
- Professional flavor notes, trimmed if needed
- A few `flavor_tags`
- Putaoso branding
- Detail page URL as small text

Fields intentionally excluded from v1:

- Palate chart
- Price range
- Pairing suggestions
- “Avoid” note
- Representative regions
- Bottle recommendations

These are useful on the page, but they would make the card feel more like a dense cheat sheet than a collectible card.

### 4. Visual Direction

1. Card style:
   - collectible field-guide card
   - refined, editorial, wine-guide-like
   - should feel like part of Putaoso, not a generic social template

2. Recommended visual language:
   - field-guide specimen card
   - restrained typographic hierarchy
   - small taxonomy-like metadata
   - paper texture / rule lines / serial number details
   - flavor terms as curated labels, not loud badges

3. Scene treatment:
   - Prefer reusing the existing page’s SVG scene in the card.
   - If the full scene is too dense at card scale, use a simplified crop, silhouette, or scene-connected accent.
   - The card should feel visually related to the variety detail page.

4. Recommended type hierarchy:
   - English name remains the large collectible title, matching the current detail page.
   - Chinese name gets strong secondary placement.
   - Quote / tagline and flavor notes can be Chinese-first for readability.

5. Recommended type-aware accents:
   - red varieties: wine accent
   - white varieties: muted gold accent
   - rosé varieties: soft rose accent
   - keep the overall paper/card system consistent across all types

### 5. Interaction Details

1. Button label:
   - `分享`

2. Recommended flow:
   - button opens modal
   - modal shows the actual card preview
   - user clicks `下载 PNG`
   - button shows a rendering state while the PNG is generated

3. v1 should not include style or size controls.

4. Recommended mobile behavior:
   - open modal
   - show scaled preview
   - keep download/share actions easy to reach
   - keep v1 download-only; do not add Web Share API yet

5. File name:
   - `putaoso-{slug}.png`

### 6. Technical Comfort

1. Dependency decision:
   - add `html-to-image` for DOM-to-PNG export

2. Preview fidelity:
   - The generated PNG should match the modal preview as closely as practical.
   - Minor browser text-rendering differences are acceptable if the final PNG still looks polished.

3. Offline behavior:
   - No special offline support for v1.
   - Once the page and fonts are loaded, export should work without additional API calls.

4. Browser targets:
   - Modern Chrome desktop/mobile
   - Safari desktop/mobile where feasible
   - WeChat in-app browser is a nice-to-test target, not a v1 blocker

## Recommended V1 Scope

Ship one high-quality fixed-format card:

- Size: `1080 × 1350` portrait, suitable for Xiaohongshu / Instagram feed and easy mobile viewing.
- Content:
  - `N°`
  - English name
  - Chinese name
  - type + origin
  - hero quote or card tagline
  - flavor notes
  - flavor tags
  - Putaoso branding
  - `putaoso.com/{slug}` as small text
- Interaction:
  - share button near the detail hero name
  - native `<dialog>` modal
  - preview card scaled to viewport
  - primary action: `下载 PNG`
  - secondary action: close
- Export:
  - client-side PNG generation from the preview DOM
  - file name: `putaoso-{slug}.png`

This keeps v1 focused while leaving room for future variants.

## Technical Plan

### Architecture

Use a small Astro component for the markup and vanilla JS for behavior:

| Piece | Responsibility |
|---|---|
| `src/components/ShareCard.astro` | Button, dialog, card preview DOM, inline JSON payload |
| `src/pages/[slug].astro` | Pass current variety data into `ShareCard` |
| `src/styles/global.css` | Share button, dialog, card, responsive preview styles |
| `package.json` | Add `html-to-image` dependency |

### Export Strategy Options

Use `html-to-image`.

Why:
- Fastest implementation.
- Preview DOM and exported PNG can match closely.
- Keeps design in HTML/CSS, which fits the current project.

Known risks:
- Font loading and SVG details need testing in Safari / WeChat browser.
- Export should wait for `document.fonts.ready`.
- Card colors should be fixed in the card CSS instead of inheriting dark mode.

## Implementation Steps

### Phase 1: Product Decisions

- Product choices are locked:
  - button label is `分享`
  - v1 is download-only
  - card should reuse or visually echo the current variety SVG scene

### Phase 2: Component Skeleton

- Create `ShareCard.astro`.
- Add a share button near the hero title in `src/pages/[slug].astro`.
- Render a native `<dialog>` with preview and actions.
- Ensure keyboard close, backdrop click, and focus behavior are reasonable.

### Phase 3: Card Design

- Build the fixed-format share card DOM.
- Use existing site tokens and fonts.
- Add type-aware accents for red / white / rosé.
- Include the existing variety scene or a simplified scene-connected treatment.
- Make the card preview scale down responsively inside the modal.

### Phase 4: PNG Download

- Install and use `html-to-image`.
- Wait for `document.fonts.ready` before rendering.
- Generate PNG blob/data URL.
- Download with a stable filename.
- Disable the download button while rendering.
- Show a lightweight error message if export fails.

### Phase 5: Verification

- Run `npm run build`.
- Test one red, one white, and one rosé detail page.
- Check desktop and mobile modal layout.
- Verify exported PNG dimensions and visual fidelity.
- Test long names and aliases.
- Test at least Chrome and Safari; WeChat browser if that target matters.

## Edge Cases

| Case | Expected handling |
|---|---|
| Very long English name | wrap elegantly without overflow |
| Very long Chinese name | keep readable hierarchy |
| Missing aliases | omit alias row |
| White / rosé varieties | use correct type label and accent color |
| Export clicked before fonts load | wait for fonts |
| PNG generation fails | show error and keep modal open |
| Small mobile viewport | scale preview, keep actions sticky / visible |
| Dark mode system preference | exported card should use intentional fixed colors, not accidental dark mode |

## Future Extensions

- Multiple visual templates.

- Generate Open Graph image assets at build time.
